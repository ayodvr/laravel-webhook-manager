# Contributing to Laravel Webhook Manager

Thank you for considering contributing to Laravel Webhook Manager! We appreciate your time and effort.

## Ways to Contribute

### 1. Reporting Issues
- Use GitHub issues to report bugs
- Provide detailed steps to reproduce the issue
- Include environment information (PHP version, Laravel version, etc.)

### 2. Feature Requests
- Open an issue to discuss new features before implementing
- Clearly describe the use case and benefits

### 3. Code Contributions
- Fork the repository
- Create a feature branch from `master`
- Make your changes
- Ensure tests pass
- Submit a pull request

## Development Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/ayodvr/laravel-webhook-manager.git
   cd laravel-webhook-manager
   ```

2. **Install dependencies**
   ```bash
   composer install
   ```

3. **Run tests**
   ```bash
   ./vendor/bin/phpunit tests/Feature/WebhookTest.php
   ```

## Code Standards

- Follow PSR-12 coding standards
- Write clear, concise commit messages
- Add tests for new features
- Update documentation as needed

## Pull Request Process

1. Ensure your code passes all tests
2. Update the README if you're adding new features
3. Submit a pull request with a clear description
4. Wait for review and address any feedback

## Testing

- All contributions must include appropriate test coverage
- Run the full test suite before submitting
- New features should have comprehensive tests

## Questions?

If you have questions or need help, feel free to open an issue or reach out to the maintainers.
